## Theory
In formal language theory, a **Regular Expression (Regex)** is a sequence of characters that defines a search pattern, effectively representing a **Regular Language**. These languages are the lowest level in the Chomsky hierarchy and can be recognized by Finite Automata. 

A regex is built from an alphabet $\Sigma$ using three fundamental operations:
1.  **Concatenation**: Joining two strings together.
2.  **Alternation ($|$|)**: Choosing between two or more patterns (the union of sets).
3.  **Kleene Star ($*$|)**: Representing zero or more repetitions of a pattern.

In this implementation, we go beyond simple matching. We treat the regex as a generative grammar. By parsing the string into an **Abstract Syntax Tree (AST)**, we can recursively traverse the structure to expand the language into its constituent words. Since regular expressions with quantifiers like $*$ or $+$ can define infinite languages, we apply a "bounded expansion" to keep the output finite and manageable.

## Implementation Description

* **AST Node Definitions**: Using Python's `dataclasses`, I defined a hierarchy of nodes representing the building blocks of a regex. `LiteralNode` handles single characters, while `ConcatNode`, `AlternationNode`, and `RepeatNode` represent the logical operations performed on sub-expressions.

```python
@dataclass(frozen=True)
class RepeatNode(RegexNode):
    node: RegexNode
    min_times: int
    max_times: int
    label: str
```

* **The Regex Parser**: This class implements a recursive descent approach, breaking the regex into "expressions" (alternations), "terms" (concatenations), and "factors" (atoms with quantifiers). It manages a pointer through the source string, skipping whitespace and handling nested parentheses to ensure correct mathematical precedence.

```python
def parse_expression(self) -> RegexNode:
    branches = [self.parse_term()]
    while self.peek() == "|":
        self.advance()
        branches.append(self.parse_term())
    return branches[0] if len(branches) == 1 else AlternationNode(tuple(branches))
```

* **The Expansion Logic**: The `expand_node` function is the heart of the generation process. It recursively visits each node; for instance, a `ConcatNode` performs a Cartesian product of the sets of words generated by its children, while an `AlternationNode` performs a set union.

```python
if isinstance(node, ConcatNode):
    combined: Set[str] = {""}
    for part in node.parts:
        part_words = expand_node(part, recorder)
        combined = {p + s for p in combined for s in part_words}
    return combined
```

* **Bounded Repetition**: To prevent infinite loops during expansion of the Kleene star ($*$) and the plus quantifier ($+$), I implemented a `MAX_UNBOUNDED_REPETITIONS` constant. This allows the system to simulate the "flavor" of the infinite language while remaining within the bounds of physical memory.
